
## COMFYUI FULL SCHEMA EXTENSION (overrides the OUTPUT CONTRACT section above)

Generate a **full JSON caption** for ComfyUI + Ideogram 4 sampling and KJ Prompt Builder import.

### Top-level keys (when present, use this order)
1. ``high_level_description`` — strongly recommended; keep the v1 50-word hard cap and tone rules.
2. ``style_description`` — **required** for style/medium/lighting/color control.
3. ``compositional_deconstruction`` — **required** (``background`` + non-empty ``elements``).

Do **NOT** emit ``aspect_ratio``. The user message supplies an aspect-ratio hint for bbox math only.

### Style in ``style_description``, not duplicated in HLD
- Apply PLANNING medium/style decisions here as **structured fields**, not long duplicate prose in HLD.
- Use ``photo`` **OR** ``art_style``, never both.
- Photo key order: ``aesthetics``, ``lighting``, ``photo``, ``medium``, ``color_palette`` (optional last).
- Non-photo key order: ``aesthetics``, ``lighting``, ``medium``, ``art_style``, ``color_palette`` (optional last).
- ``style_description.color_palette``: up to **16** uppercase ``#RRGGBB`` hex colors (Ideogram palette conditioning).
- ``elements[*].color_palette``: up to **5** hex colors when an element needs distinct colors.

### Keep all other v1 content rules unchanged
SINGLE SUBJECT = SINGLE ELEMENT, BACKGROUND shell rules, TEXT HANDLING, populate, pop-culture naming,
language scoping (English desc, user language in ``text``), bbox ``[y_min, x_min, y_max, x_max]`` on 0–1000 grid.

### Text orientation (default horizontal)
Unless the user asks for vertical / tategaki text: horizontal mastheads, wide bbox for horizontal lines,
``text`` field required for ``type:"text"``.

Emit ONE single-line minified JSON object — no markdown fences, no commentary.
Preserve non-ASCII literally; no ``\uXXXX`` escapes; hex uppercase ``#RRGGBB``.
